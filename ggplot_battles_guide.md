# ggplot battles 入門ガイド & ゼミ演習プラン

サイト: https://www.ggplotbattles.dev/

---

## 1. ggplot battles とは(要点まとめ)

- **目的**: 完成済みの ggplot2 グラフ(ターゲット画像)を見て、それをできるだけ忠実に **R コードで再現する** ブラウザゲーム型の練習サイト。
- **仕組み**:
  - 「Battle(バトル)」と呼ばれる課題を選ぶ
  - ブラウザ上のエディタで R コードを書く
  - 自分のプロットとターゲット画像が比較され、**類似度スコア**が即座に表示される
- **環境構築が不要**: [webR](https://docs.r-wasm.org/webr/latest/)(WebAssembly 版の R)で動くため、**R や RStudio をインストールせずに** ブラウザだけで完結する。
- **難易度**: Easy〜Hard まで幅広く、難易度・プロットの種類・進捗でフィルタできる。
- **教材としての特徴**: 「お手本を見て逆算する」形式なので、`ggplot2` の文法(geom、aesthetics、scale、theme など)を実践的に学べる。
- **コントリビュート可能**: 課題は GitHub リポジトリ([MikeLydeamore/ggplot2-battles](https://github.com/MikeLydeamore/ggplot2-battles))で管理されており、自己完結した R ファイルを追加すれば新しいバトルを作れる。
- **作者**: Michael Lydeamore 氏(モナシュ大学)。

> 一言でいうと: **「写経ではなく再現」で ggplot2 を鍛える、ブラウザ完結型のトレーニング場**。

## 2. Hello World 編:一番簡単なバトルをやってみよう

サイトの看板課題になっている **「Penguin Scatters」**(ペンギンの散布図)が、事実上の Hello World です。`palmerpenguins` データを使った基本の散布図を再現します。

### 手順

1. https://www.ggplotbattles.dev/ を開く
2. 「Choose a battle」から難易度 **Easy** の散布図(Penguin Scatters など)を選ぶ
3. 右側のターゲット画像を観察する
   - **x 軸・y 軸は何の変数か?**
   - **色分けはあるか?(種ごと?)**
   - **タイトルや軸ラベルは?**
4. エディタに以下のようなコードを書いて Run する

```r
library(ggplot2)
library(palmerpenguins)

ggplot(penguins, aes(x = flipper_length_mm, y = body_mass_g)) +
  geom_point()
```

5. スコアを確認 → ターゲットとの違い(色、ラベル、テーマ)を探して修正

```r
# 例:種ごとに色分けを追加
ggplot(penguins, aes(x = flipper_length_mm, y = body_mass_g,
                     colour = species)) +
  geom_point()
```

### Hello World で身につくこと

- `ggplot(データ, aes(...))` + `geom_*()` という **レイヤー構造の基本**
- `aes()` に変数をマッピングする感覚(x, y, colour)
- 「見た目の差分 → コードの差分」に変換する観察力(このサイトの核心)

## 3. ゼミ演習プラン(大学2年生向け・全3回)

対象: 統計・データ分析の入門を終えた学部2年生
形式: 各回90分、ペアまたは個人でブラウザ演習。**インストール不要**なので PC 教室でも自分の PC でも実施可能。

---

### 第1回:ggplot2 の文法とバトル入門

**目標**: レイヤー文法を理解し、Easy バトルを1つクリアする。

| 時間 | 内容 |
|---|---|
| 0–20分 | 講義: グラフィックスの文法(data / aes / geom / scale / theme) |
| 20–50分 | 全員で Penguin Scatters を再現(上記 Hello World) |
| 50–80分 | 各自 Easy バトルをもう1つ選んで挑戦(棒グラフ・ヒストグラム系推奨) |
| 80–90分 | スコア共有・つまずきポイントの振り返り |

**キーワード**: `geom_point`, `geom_bar` / `geom_col`, `geom_histogram`, `labs()`

**宿題**: Easy バトルを1つ、スコア90%以上でクリアしてスクリーンショット提出。

---

### 第2回:見た目を制御する(scale・facet・theme)

**目標**: ターゲット画像の「細部」を再現できるようになる。

| 時間 | 内容 |
|---|---|
| 0–15分 | 前回の宿題レビュー:高スコアの人のコードを共有 |
| 15–35分 | 講義+デモ: `scale_colour_manual()` / `scale_*_continuous()`、`facet_wrap()`、`theme_minimal()` などの組み込みテーマ、`theme()` による凡例位置・グリッド調整 |
| 35–75分 | Medium バトルに挑戦(色指定やファセットを含むもの) |
| 75–90分 | 「どの要素がスコアに効いたか」をペアで議論 |

**キーワード**: `scale_*`, `facet_wrap` / `facet_grid`, `theme_*`, `coord_flip`, `labs(title, subtitle, caption)`

**宿題**: Medium バトル1つ + 「ターゲットと自分のプロットの差分を3点言語化する」ミニレポート(半ページ)。

---

### 第3回:データ変換とチャレンジ大会

**目標**: 作図前のデータ加工(dplyr)を組み合わせ、難しめのバトルに取り組む。

| 時間 | 内容 |
|---|---|
| 0–20分 | 講義: 作図のためのデータ整形 —— `group_by() + summarise()`、`filter()`、因子の並べ替え(`forcats::fct_reorder`) |
| 20–60分 | Medium〜Hard バトルに挑戦(集計を要するもの、順序付き棒グラフなど) |
| 60–85分 | **ミニ大会**: 制限時間25分で指定バトルのスコアを競う。同点はコードの簡潔さで判定 |
| 85–90分 | まとめ: 実データでの「良い可視化」への橋渡し(次のステップとして自分のデータで作図する課題を予告) |

**キーワード**: `dplyr` パイプライン, `fct_reorder`, `geom_line` + `group`, 注釈(`annotate`, `geom_text`)

**最終課題(レポート)**: 任意のバトル1つについて、(1) 最終コード、(2) スコア、(3) 再現の過程で学んだ ggplot2 の機能を3つ説明、をまとめて提出。

## 4. 教員向けメモ

- **採点のしやすさ**: 類似度スコアが自動で出るため、達成度の客観指標として使える。
- **環境トラブルほぼゼロ**: webR のおかげでインストール指導が不要。初回の立ち上がりが速い。
- **注意点**: webR の初回ロードに少し時間がかかることがあるので、授業前にページを開いておくよう指示するとよい。
- **発展**: ゼミの成果として、学生にオリジナルのバトルを作らせて GitHub リポジトリに PR を出させると、再現「する側」から「させる側」への学びの転換になる。
